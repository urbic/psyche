массив

Операнды: 

  • массив: array
  • ключ: object
  • компаратор: proc
  • индекс: integer
  • результат: boolean

Ищет заданный ключ в массиве, используя алгоритм двоичного поиска. Если массив содержит несколько элементов, равных (в соответствии с компаратором) заданному ключу, индекс может принадлежать любому из них.

Примеры: 

[ 0 10 20 30 ] 15 { cmp } binarysearch	▶ 2 false
[ 0 10 20 30 ] 100 { cmp } binarysearch	▶ 4 false
[ 0 10 20 30 ] 10 { cmp } binarysearch	▶ 1 true

Ошибки: 

  • stackunderflow
  • typecheck

Смотри также: cmp, sort.

4.2.75. eq

Возвращает true, если объект1 равен объекту2, и false в противном случае. 

объект1
объект2
eq
результат

Операнды: 

  • объект1, объект2: object
  • результат: boolean

Понятие равенства объектов зависит от их типов. Объекты одного и того же простого типа считаются равными, если совпадают их значения. Объекты числовых типов считаются равными, если представляют одно и то же математическое число. Объекты типа stringy считаются равными, если их строковые значения совпадают. Объекты прочих сложных типов равны, только если они идентичны.

Следует принимать во внимание, что при математических вычислениях возможна потеря точности из-за округления, так что ожидаемое равенство может быть нарушено.

Примеры: 

3. 3 eq					▶ true	# Равные вещественное и целое числа
1 0 complex 1. eq		▶ true	# Равные комплексное и вещественное числа
"xyz" "xyz" eq			▶ true	# Строки с одинаковыми значениями равны
/xyz "xyz" eq			▶ true	# Имя и строка с одинаковыми значениями равны
[ 0 1 2 ] dup eq		▶ true	# Два экземпляра одного и того же массива равны
[ 0 1 2 ] [ 0 1 2 ] eq	▶ false	# Отдельные экземпляры массивов не равны
1 77777777 div 77777777 mul 1 eq	▶ false	# Равенство нарушено из-за потери точности

Ошибки: 

  • stackunderflow

Смотри также: ne.
