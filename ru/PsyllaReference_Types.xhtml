В языке Psylla имеется гибкая и расширяемая система типов. Типы реализованы как классы или интерфейсы языка Java. Каждый объект в Psylla представлен как объект некоторого класса, реализующий интерфейс PsyObject.

Расширяемость системы типов достигается за счёт программирования на Java новых производных от PsyObject классов. Создание новых типов средствами Psylla невозможно.

Большинство операторов Psylla рассчитывают на определённый тип своих операндов. Так, оператор сложения add может применяться к числам, но не к строкам.

Некоторые операторы полиморфны, то есть способны менять свой смысл в соответствии с типом своих операндов. К примеру, тот же оператор add для целочисленных операндов возвращает результат целого типа (за исключением того случая, когда сумма выходит за границы целого типа; тогда результат имеет вещественный тип). Если один из операндов целый, а другой — вещественный, или же оба вещественные, результат также будет иметь вещественный тип. Если хотя бы один из операндов имеет комплексный тип, такого же типа будет результат. Другой пример: оператор or вычисляет дизъюнкцию своих операндов. Если оба операнда булевского типа, результат будет булевским, а если оба операнда целочисленные, выполняется побитовая дизъюнкция, и результат будет целым числом.
