5.1. Errors in Detail

5.1.1. concurrentmodification

Concurrent modification detected.

5.1.2. dictstackunderflow

Dictionary stack underflow. The error is generated by the end operator, if the dictionary stack contains only permanent dictionaries.

5.1.3. directorynotempty

The directory is not empty. The error is generated by the operators expecting that the directory is empty, when this condition fails.

5.1.4. fileaccessdenied

File access denied.

5.1.5. fileexists

File exists. The error is generated by the operators expecting that the file does not exist, when this condition fails.

5.1.6. filenotfound

File not found. The error is generated by the operators expecting that the file exists, when this condition fails.

5.1.7. invalidcontext

Invalid context.

5.1.8. invalidexit

Invalid exit. The error is generated by the exit operator when invoked outside the cyclic context.

5.1.9. invalidexternal

Неправильный внешний класс. Генерируется при загрузке внешнего класса или создании объекта этого класса, если класс не найден на пути поиска, не реализует интерфейс coneforest.psi.PsiObject или не имеет общего конструктора без аргументов.

5.1.10. invalidregexp

Invalid regular expression. The error is generated by the parser or by the regexp operator when the regular expression is syntactically inconsistent.

5.1.11. invalidstate

Invalid state. The error is generated if the execution of the operator is not possible with the current state of the operand.

5.1.12. ioerror

Input/output error.

5.1.13. limitcheck

Превышение установленных ограничений.

5.1.14. notdirectory

Not a directory.

5.1.15. notlink

Not a link.

5.1.16. rangecheck

The error is generated when the operand has the value outside the range acceptable for the operator.

5.1.17. securityerror

Security policy violation.

5.1.18. stackunderflow

Operand stack underflow. The error is generated when the operand stack contains fewer operands than the operator requires.

5.1.19. syntaxerror

Syntax error.

5.1.20. typecheck

Type mismatch. The error is generated if the operand has the type that is unexpected for the operator.

5.1.21. undefined

Undefined key. The error is generated by the operators accessing the dictionary by the absent key. Also, is generated by the interpreter when executing the command with the name absent in the dictionary stack.

5.1.22. undefinedresult

Undefined result. The error is generated when the result of the operation is not defined, for example, when trying to integer division by zero in the idiv operator.

5.1.23. unmatchedmark

Unmatched mark. The error is generated by the operators receiving the mark-delimited list of operands when the mark is absent in the operand stack.

5.1.24. unsupported

Unsupported operation. The error is generated by the operators trying to perform an unsupported operation.
