The Psylla language has a flexible and extendable type system. The types implemented as Java classes or interfaces. Each Ψ object is represented as an object of some class implementing the PsyObject interface.

Extensibility of the type system is based on the Java programming new classes from PsyObject. The creation of new types by means of the Psylla language is impossible.

Majority of Psylla operators expect operands of certain type. So, the addition operator add can be applied to the numbers, but not to strings.

Some operators are polymorphic, that is able to change its meaning in accordance with the type of its operands. For example, the same operator add for integer operands returns integer (except in the case where the sum is beyond the boundaries of integer type; then the result is real). If one of the operands is an integer, and the other is a real, or both are real, the result will also have a real type. If at least one of the operands has the complex type, the result will have the same type. Another example: the or operator returns the disjunction of its operands. If both operands are of boolean type, the result will be boolean, and if both operands are integer, bitwise disjunction is calculated, and the result will be integer.
